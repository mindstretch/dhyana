Dhyana · Style Guide · v1.0

Still. Warm. Alive.

The complete visual and interaction language for Dhyana. Every decision here is grounded in research and purpose — not aesthetics alone.

01 · Foundation

Philosophy

Every design decision passes through one test.

"Does this feel like taking a breath?"

If it feels like filling out a form — start over.
If it feels like opening a notification — start over.
If it feels like a dashboard — start over.
If it makes someone exhale slightly when they see it — you're there.

The Promise

When someone opens Dhyana, before they do anything, they should feel: "Someone made this for me."

After Check-In

After they check in, they should feel: "I know where I am now."

02 · Foundation

Voice & Tone

Sparse. Warm. Direct. Grounded in the body. Never judgment.

App UI — Check-in prompt

✓ Use

"Where are you right now?"

✗ Not

"Please complete your daily nervous system assessment to generate your score."

State reveal

✓ Use

"Restful Alertness. Calm and clear. Fully present. Ready."

✗ Not

"Your nervous system state has been classified as Restful Alertness (Score: 84/100)."

Email — state-aware opening

✓ Use

"You showed up running hot. Dhyana would know what to do with that."

✗ Not

"Thank you for signing up! Based on your selected state (Activated), we think you'll love our stress management features."

Push notification (Phase 3 Agent)

✓ Use

"Your nervous system is activated right now. You have 8 minutes. Here's something that will help."

✗ Not

"🔥 Don't break your streak! Check in now to maintain your 7-day wellness journey."

06 · Visual

Motion

Everything that moves in Dhyana communicates one thing: something alive. Slow is right. Nothing is instantaneous. Nothing draws attention to itself.

Orb ambient breathe 7s ease-in-out infinite · scale(1→1.022) The orb is alive
Glow breathe 6s ease-in-out infinite · opacity + scale Ambient warmth
Specular shimmer 9s ease-in-out infinite · translate XY Something under the surface
Indicator placement 350ms cubic-bezier(0.34,1.56,0.64,1) Spring — arrives, doesn't snap
State reveal 500ms ease · opacity + translateY(5px→0) Recognition takes a moment
Content fade-up 600ms ease · opacity + translateY(8px→0) Form, reveal line
Ripple expansion 1400ms · 3 rings · staggered 180ms Touch had an effect
Volume control fade-in 1800ms ease · 2s delay · opacity 0→1 Doesn't distract on load
Color transitions 1400ms ease State color shifts, glow changes

07 · Visual

The Orb

The orb is not a UI element. It is the product. It only appears in the context of check-in — never decoratively.

Mobile sizemin(260px, 58vw, 44vh)
Desktop sizemin(300px, 40vw, 42vh)
Shadow0 20px 56px rgba(0,0,0,0.5)
Border1px rgba(255,255,255,0.05)
Breathe7s ease-in-out infinite

The orb maps the nervous system state space. Upper-right quadrant is Restful Alertness (warm amber). Upper-left is Activated (coral). Lower-left is Depleted (slate blue). Lower-right is Dysregulated (mauve). Center is Regulated (teal). The gradient is always present — state color shifts the ambient glow when a state is identified, not the orb itself.

13 · Application

Do's & Don'ts

What Dhyana never does is as important as what it does.

✓ Do

One primary action per screen

✗ Don't

Multiple CTAs competing for attention

✓ Do

State colors on state-identified moments only

✗ Don't

State colors as decorative UI accents

✓ Do

Generous whitespace — when in doubt, add more

✗ Don't

Tight layouts that feel like information architecture

✓ Do

DM Sans at weight 300–400 only

✗ Don't

Bold or heavy type — pressure is the opposite of Dhyana

✓ Do

Slow, purposeful animation — everything alive, nothing showy

✗ Don't

Fast transitions or playful bounce physics

✓ Do

Non-judgmental language — there is no bad state

✗ Don't

Streaks, badges, or anything that implies you can fail
